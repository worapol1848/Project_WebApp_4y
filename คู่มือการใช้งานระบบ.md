# คู่มือการใช้งานระบบ Velin Inventory & E-Commerce

เอกสารฉบับนี้เป็นคู่มือการใช้งานฉบับสมบูรณ์สำหรับ **ระบบ Velin Inventory & E-Commerce** ซึ่งครอบคลุมทุกฟังก์ชันการทำงานทั้งในส่วนของลูกค้า (Customer Frontend) และส่วนของผู้ดูแลระบบ (Admin Dashboard)

---

## 1. การเริ่มต้นและเปิดระบบ (Starting the System)

เพื่อให้สะดวกต่อการทำงานและการทดสอบ ระบบได้เตรียม Script สำหรับเปิดเซิร์ฟเวอร์ไว้ให้แล้ว
ไปที่โฟลเดอร์หลักของโปรเจกต์ (Velin Web App) และดับเบิลคลิกไฟล์ต่อไปนี้:

*   **`start_app.bat`**: ใช้สำหรับเปิด Backend และ Frontend ขึ้นมาพร้อมกันในหน้าต่างเดียว
*   **`start_dual_sessions.bat`**: ใช้สำหรับเปิดระบบแบบ **Dual Session** (มีประโยชน์มากสำหรับการทดสอบ) โดยระบบจะเปิด:
    1.  Backend (Port 5000)
    2.  Frontend A (Port 5173) - สำหรับเปิดหน้า Admin ทิ้งไว้
    3.  Frontend B (Port 5174) - สำหรับเปิดหน้า Customer ทิ้งไว้
    *(ทำให้คุณสามารถทำรายการสั่งซื้อในฝั่งลูกค้า และดูอัปเดตแบบ Real-time ในฝั่งแอดมินได้โดยไม่ต้องสลับบัญชีไปมา)*

---

## 2. กลุ่มผู้ใช้งาน (User Roles)

ระบบนี้รองรับกลุ่มผู้ใช้งานหลักๆ 4 ระดับ ได้แก่:
1.  **Guest (ผู้เยี่ยมชม):** สามารถเข้าชมเว็บไซต์ ดูรายละเอียดสินค้า และค้นหาสินค้าได้ (หากต้องการสั่งซื้อต้องสมัครสมาชิก)
2.  **User (ลูกค้า):** สามารถทำการสั่งซื้อสินค้า, เปลี่ยนแปลงข้อมูลส่วนตัว, ติดตามสถานะคำสั่งซื้อ, และบันทึกสินค้าที่ชอบ (Wishlist) ได้
3.  **Admin (ผู้ดูแลระบบ):** สามารถเข้าถึงระบบหลังบ้าน จัดการสินค้า (Inventory), อัปเดตสถานะคำสั่งซื้อ, และดู Dashboard ข้อมูลสรุปยอดขายได้
4.  **Superadmin (ผู้ดูแลระบบสูงสุด):** มีสิทธิ์เหมือน Admin ทุกประการ แต่เพิ่มเติมคือสิทธิ์ในการจัดการผู้ใช้ระดับแอดมินด้วยกัน และ **ต้องลงทะเบียน/เข้าสู่ระบบผ่านการสแกนใบหน้า (Face Scanner Biometric)** เท่านั้น

---

## 3. ส่วนการใช้งานของลูกค้า (Customer / User Portal)

### 3.1 การเข้าสู่ระบบ / การสมัครสมาชิก (Login & Register)
*   **สมัครสมาชิก:** ผู้ใช้ใหม่สามารถสมัครสมาชิกเพื่อรับสิทธิ์ทำการสั่งซื้อ โดยข้อมูลทั้งหมดจะถูกจัดเก็บอย่างปลอดภัย
*   **การตั้งค่าภาษา (Isolated Language):** ระบบมีการแยกการตั้งค่าภาษาต่อผู้ใช้งาน 1 คนอย่างชัดเจน (เมื่อสลับ User ภาษาของระบบจะปรับตามการตั้งค่าล่าสุดของ User นั้นๆ ทันที)

### 3.2 การค้นหาและเลือกซื้อสินค้า (Product Catalog)
*   หน้า **"All Products"** ออกแบบสไตล์ Minimalist สะอาดตา (Shadow-free) 
*   **Brand Filter Bar:** มีแถบกรองยี่ห้อ (เช่น Nike, Jordan, etc.) พร้อม Logo แบรนด์ ค้นหาสินค้าได้ง่าย
*   **Carousel / Slider:** ระบบจะแสดงรายการสินค้าแนะนำ หรือสินค้าขายดี (Best-Sellers) ในรูปแบบสไลเดอร์รูปภาพสินค้าจริง ผู้ใช้สามารถคลิกที่รูปเพื่อเข้าไปดูรายละเอียดได้

### 3.3 ระบบตะกร้าสินค้า (Shopping Cart)
*   **Member Only Cart:** ระบบบังคับให้ผู้ใช้งานต้องล็อกอินเข้าสู่ระบบก่อน จึงจะสามารถกดเพิ่มสินค้าลงตะกร้าได้ เพื่อป้องกันข้อมูลตะกร้าสินค้าสูญหาย

### 3.4 การสั่งซื้อและชำระเงิน (Checkout & Payment)
*   **ที่อยู่สำหรับจัดส่ง:** ระบบรองรับการระบุที่อยู่และการปักหมุดผ่านแผนที่ (React Leaflet)
*   **Address Snapshots:** ทุกครั้งที่ทำการสั่งซื้อสำเร็จ ระบบจะทำ **Snapshot ถ่ายรูปข้อมูลที่อยู่** ณ เวลานั้นเก็บไว้กับ Order หากในอนาคตผู้ใช้ไปแก้ไขที่อยู่ใน Profile ข้อมูลในฝั่งของใบเสร็จ/คำสั่งซื้อที่เคยซื้อไปแล้วจะไม่ถูกเปลี่ยนแปลง (ป้องกันความผิดพลาดทางบัญชีและประวัติการขนส่ง)
*   **ใบเสร็จ PDF:** เมื่อการสั่งซื้อสมบูรณ์ ระบบสามารถรันและดาวน์โหลดใบเสร็จ/ใบสรุปรายการสั่งซื้อออกมาเป็นไฟล์ PDF ได้

### 3.5 ข้อมูลส่วนตัว (Profile & History)
*   **My Orders:** ติดตามสถานะคำสั่งซื้อในอดีตทั้งหมด (เช่น Pending, Shipped, Delivered)
*   **Wishlist:** บันทึกสินค้าที่ชื่นชอบเก็บไว้ดูทีหลัง
*   **Comments/Reviews:** สามารถรีวิวสินค้าและให้คะแนนสินค้าที่เคยซื้อได้

## 4. ส่วนการใช้งานของผู้ดูแลระบบ (Admin / Superadmin Portal)

### 4.1 การเข้าสู่ระบบด้วยใบหน้า (Face Scanner Biometric)
*   **ฟังก์ชันนี้สงวนไว้เฉพาะบทบาท `Superadmin` เท่านั้น**
*   ในหน้าล็อกอินของ Superadmin ระบบปฏิบัติการจะเรียกใช้กล้องหน้า (Webcam) โดยอาศัยเทคโนโลยี `face-api`
*   ใช้สำหรับยืนยันตัวตนขั้นสูงสุด ป้องกันการถูกขโมยรหัสผ่าน (การลงทะเบียนใบหน้าครั้งแรก ทำได้ที่หน้า SuperAdminManage)

### 4.2 ระบบ Dashboard และ Analytics
*   **ภาพรวมทั้งหมด (AdminDashboard):** แสดงกราฟยอดขาย, รายได้, จำนวนคำสั่งซื้อใหม่, และผู้ใช้งานใหม่ อัปเดตข้อมูลเพื่อให้แอดมินวิเคราะห์ทิศทางธุรกิจได้ทันที
*   **รายงานสินค้ายอดฮิต (AdminBestSellers):** ติดตามได้ว่าสินค้าใดขายดี สินค้าใดกำลังจะหมดสต็อก (Low Stock Alerts)
*   **AdminRevenue:** เจาะลึกรายได้รายเดือน/รายปี พร้อมกราฟอย่างละเอียด (Recharts)

### 4.3 การจัดการสินค้าคงคลัง (Inventory & Product Management)
*   **AdminProducts / AdminProductDetail:** 
    *   สามารถ **เพิ่ม** สินค้าใหม่ พร้อมตั้งระดับสต็อก ราคา และเลือกหมวดหมู่ยี่ห้อ (Brand)
    *   สามารถ **แก้ไข/ลบ** ข้อมูลสินค้า หรือทำส่วนลด
    *   **อัปโหลดรูปภาพ:** ระบบรองรับการอัปโหลดไฟล์ภาพสินค้าและนำไปแสดงผลที่หน้าบ้านในรูปแบบ Dynamic

### 4.4 การจัดการคำสั่งซื้อ (Order Management)
*   **AdminOrders:** แสดงรายการคิวคำสั่งซื้อที่เข้ามาทั้งหมด
*   ตรวจสอบรายละเอียดใบสั่งซื้อ, สินค้าทั้งหมดค้างส่ง, และที่อยู่ของลูกค้า (จาก Snapshot)
*   แอดมินสามารถกด **อัปเดตสถานะ** เช่น เปลี่ยนจาก "รอตรวจสอบ" เป็น "จัดส่งแล้ว" ซึ่งจะสะท้อนไปที่หน้า My Orders ของลูกค้าทันที

### 4.5 ระบบติดตามและบันทึกประวัติ (System Logs)
*   **AdminLogs:** บันทึกประวัติการกระทำทั้งหมดในหลังบ้าน (Audit Logging) อ้างอิงว่าแอดมินคนไหนทำอะไร เมื่อไหร่ ตัวอย่างเช่น "Admin A เปลี่ยนราคาสินค้า B", "Admin C ลบสินค้า X ออกจากระบบ" เพื่อการตรวจสอบย้อนหลังที่โปร่งใส

## 5. สรุปความโดดเด่นทางเทคนิคที่ควรรู้ (Technical Highlights)

*   **ความปลอดภัยของข้อมูลลูกค้า:** ด้วยระบบ Address Snapshot ทำให้มั่นใจได้ว่าข้อมูลประวัติการทำธุรกรรมจะสมบูรณ์และตรวจสอบได้เสมอ
*   **Localization Scope:** ไม่ว่าจะมีผู้ใช้กี่คนเข้าใช้งานด้วยเครื่องคอมพิวเตอร์เดียวกัน ภาษาและ Environment จะไม่ทับซ้อนกัน ข้อมูลทั้งหมดจะถูกรีต (Reset / Load) ทันทีที่สลับ account
*   **Secure Cart Workflow:** การบังคับล็อกอินก่อนนำสินค้าลงตะกร้า ช่วยลดปัญหาขยะข้อมูลในระบบ (Orphan Carts) และหลีกเลี่ยงประสบการณ์แย่ๆ จากข้อผิดพลาดของการสูญหายของ LocalStorage
